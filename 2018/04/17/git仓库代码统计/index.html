git仓库代码统计

发表于 2018-04-17 更新于 2022-06-22 分类于 编程语言
本文字数： 1.7k 阅读时长 ≈ 2 分钟

虽然以代码行数来衡量项目或者程序员并不是一件靠谱的事，但是从统计角度看趋势对于技术管理人员还是很有帮助的！推荐一个比较好用的git仓库代码统计工具：git_stats，它用于按git提交人、提交次数、修改文件数、代码行数、注释量在时间维度上进行统计，亦可按各文件类型进行简单的统计，非常方便。实际上，这么多功能通常都是用WEB在多个页面上显示的，git_stats也是如此，它需要你先安装好ruby以生成基础的页面，再用gem安装好git_stats，最后用git_stats一条语句即可生成展示页面。这些静态页面如需共享，那么搭个nginx显示静态页面即可。
